## 存储对账记录

> R17（AGENTS.md §8.3，机读硬门禁）：**开发窗口批次集成测试阶段**须满足 `batchStorageReconPresent`（`checkBatchStorageReconciliationReport`）。本章节须含真实数据行；**场景类型**在未豁免 R14 时至少一行「接口」/API **且介质非「不适用」**，未豁免 E2E 时至少一行「E2E」/UI **且介质非「不适用」**；项目未整体豁免时**至少一条适用（真实对账）行**。每行**关联任务包 / 对账方式 / 预期存储结果 / 实际存储结果 / 是否通过**须非空；**存储介质**须为可识别类别（数据库 / 文件 / 缓存 / 对象存储 / 其他 / 不适用）；介质为「其他」时**备注**须写明具体系统；介质为「不适用」时**备注**须说明该任务包无业务数据写入的理由。进度列表中已完成批次测试的任务包编号须全部出现在「关联任务包」列（按批次覆盖，首批填过不能代替后续批次；「不适用」行可计入覆盖）。**批次内确有任务包不涉及业务数据写入时**，仍须为该任务包补一行：「存储介质」填「不适用」，「备注」注明「本任务包无业务数据写入，不适用对账」；「对账方式 / 预期存储结果 / 实际存储结果 / 是否通过」可填「不适用」。**不得**用「其他」表示无写入，**不得**编造虚假对账，**不得**仅靠「不适用」行代替真实对账或项目级豁免。空章节、缺列、缺适用分类型行、仅有不适用行、介质不合法、描述列空、其他/不适用无备注或任务包未覆盖 → stop 门禁 R17 followup。无业务数据持久化可走双要素豁免（`storageReconciliationApplicability:"n/a"` + 用户确认）。对账查验语义仍由 QE/PM 审查（§8.4）。
